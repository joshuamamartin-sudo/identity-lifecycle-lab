# Identity Lifecycle Lab

This project is an IAM portfolio lab focused on the identity lifecycle: onboarding, access changes, and offboarding.

The goal is to show how user access can be requested, approved, documented, reviewed, and removed using a clear and repeatable process.

## Scenario

A fictional company needs a basic IAM process for managing employee access across common business systems.

This lab covers three common identity lifecycle events:

* New employee onboarding
* Department or role change
* Employee offboarding

## IAM Concepts Demonstrated

* User provisioning
* User deprovisioning
* Access requests
* Manager approval
* Least privilege
* Role-Based Access Control
* Access review
* Account disablement
* Documentation of access changes

## Planned Lab Files

* New hire access request
* Department transfer access change
* Offboarding checklist
* Access review example
* Least privilege notes

## Notes

This is a learning project using fictional users, fictional systems, and fictional company data. No real employee, company, or credential information is included.
